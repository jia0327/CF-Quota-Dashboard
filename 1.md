## 1. 核心架构（非常重要）

- **单 Worker 部署**：只在一个 Cloudflare 账号上部署 Worker。
- **多账号管理**：被监控账号存储在 **KV**，不通过 `wrangler.toml` `[env]`。
- **动态添加**：`/admin` 输入 Account ID、API Token、Account Name → KV。
- **定时任务**：Cron `0 */6 * * *`，循环 KV 账号抓取用量。

## 2. 技术栈

| 层级 | 选型 |
|------|------|
| Runtime | Cloudflare Workers + Hono.js |
| Storage | KV：`ACCOUNTS`、`QUOTA_SNAPSHOT`、`NOTIFICATION_CHANNELS`、`session:*` |
| Frontend | Vanilla JS + Tailwind CDN + 自研 CSS（Assets binding） |
| Cron | `0 */6 * * *` |

## 3. 数据结构设计 (KV)

### 3.1 账号配置 (Key: `ACCOUNTS`)

```json
[
  {
    "id": "uuid-account-a",
    "name": "主账号",
    "accountId": "cf_account_id_here",
    "apiToken": "cf_api_token_here",
    "enabled": true
  }
]
```

> API 返回时 `apiToken` 掩码显示。编辑时若提交掩码值则保留原 Token。

### 3.2 配额快照 (Key: `QUOTA_SNAPSHOT`)

```json
{
  "lastUpdated": "2026-01-15T10:00:00Z",
  "accounts": [
    {
      "accountId": "cf_account_id_here",
      "accountName": "主账号",
      "status": "ok",
      "lastCheckTime": "2026-01-15T10:00:00Z",
      "quotas": {
        "workers_requests": { "used": 12000, "limit": 100000, "pct": 12, "unit": "requests", "period": "daily", "label": "Workers Requests", "available": true },
        "pages_requests": { "used": 5000, "limit": 100000, "pct": 5, "unit": "requests", "period": "daily", "label": "Pages Functions Requests", "available": true }
      }
    }
  ]
}
```

完整 28 项指标 key 见 `worker/src/free-tier-limits.ts`。

### 3.3 通知渠道 (Key: `NOTIFICATION_CHANNELS`)

```json
[
  {
    "id": "uuid-channel-1",
    "type": "wecom",
    "name": "主告警群",
    "enabled": true,
    "config": { "webhookUrl": "https://..." }
  }
]
```

类型：`wecom` | `feishu` | `dingtalk` | `webhook` | `telegram` | `email`

### 3.4 QuotaMetric 字段

| 字段 | 说明 |
|------|------|
| `used` | 当前用量 |
| `limit` | 免费额度上限 |
| `pct` | 使用百分比 (0–100) |
| `unit` | 单位 |
| `period` | `daily` / `monthly` / `total` |
| `label` | 显示名称 |
| `available` | API 是否可获取；`false` 不参与告警 |
| `note` | 可选说明 |

### 4.2 API 抓取逻辑 ✅

**GraphQL**: `https://api.cloudflare.com/client/v4/graphql`

| 指标 | GraphQL 数据集 |
|------|---------------|
| Workers 请求 | `workersInvocationsAdaptive` |
| KV 操作/存储 | `kvOperationsAdaptiveGroups` / `kvStorageAdaptiveGroups` |
| D1 | `d1AnalyticsAdaptiveGroups` / `d1StorageAdaptiveGroups` |
| R2 | `r2StorageAdaptiveGroups` / `r2OperationsAdaptiveGroups` |
| Queues | `queueMessageOperationsAdaptiveGroups` |
| Workers AI | `aiInferenceAdaptiveGroups` |
| Hyperdrive | `hyperdriveQueriesAdaptiveGroups` |
| Workflows | `workflowsAdaptiveGroups` |
| Browser Run | `browserRenderingBrowserTimeUsageAdaptiveGroups` |
| Analytics Engine | `workersAnalyticsEngineAdaptiveGroups` |
| Workers Logs | `logExplorerIngestionAdaptiveGroups`（仅 bytes） |
| Durable Objects | `durableObjects*Groups` |
| Vectorize | `vectorizeQueriesAdaptiveGroups` / `vectorizeStorageAdaptiveGroups` |
| Pages Functions 请求 | `pagesFunctionsInvocationsAdaptiveGroups` |

**REST**

| 指标 | 端点 |
|------|------|
| Pages 月构建 | `GET /accounts/{id}/pages/projects` + `/deployments` |
| D1 数据库个数 | `GET /accounts/{id}/d1/database`（需 **Account → D1 → Read**） |

**常量**：`SUBREQUESTS_PER_ACCOUNT = 5`

### 4.3 告警逻辑 ✅

- 阈值：`ALERT_THRESHOLD`（默认 70）
- 条件：`metric.available && metric.pct >= threshold`
- 发送：所有 **enabled** KV 渠道并行；无 KV 渠道时回退 `WEBHOOK_URL`（隐式 wecom）
- 实现：`notifier.ts` + `channels/*.ts`

### 4.4 刷新预算 ✅

| 变量 | 默认 | 行为 |
|------|------|------|
| `ACCOUNT_CHECK_INTERVAL_MINUTES` | 20 | 距 `lastCheckTime` 不足则 `cached` |
| `MAX_EXTERNAL_SUBREQUESTS_PER_RUN` | 50（cap 50） | 超出则 `skippedByLimit` |

响应字段 `refreshStats`: `{ refreshed, failed, cached, skippedByLimit, subrequestsUsed }`

### 4.5 不可用 / 部分可用指标

| 指标 | 状态 |
|------|------|
| Workers CPU 时间 (10ms) | 无账户级日累计 API |
| Workers Logs 事件数 | 仅 bytes；`workers_logs_bytes.available = false` |
| Analytics Engine 读查询 | GraphQL 无读计数 |
| Email Routing / Sending | 无用量 API |

## 7. wrangler.toml 关键配置

```toml
name = "cf-quota-dashboard"
main = "src/index.ts"
compatibility_date = "2026-01-15"

[[kv_namespaces]]
binding = "KV"
id = "YOUR_KV_NAMESPACE_ID"

[assets]
directory = "../frontend"
binding = "ASSETS"

[triggers]
crons = ["0 */6 * * *"]

[vars]
WEBHOOK_URL = ""
ALERT_THRESHOLD = "70"
USERNAME = "admin"
ACCOUNT_CHECK_INTERVAL_MINUTES = "20"
MAX_EXTERNAL_SUBREQUESTS_PER_RUN = "50"
# FREE_TIER_LIMITS = '{"workers_requests":{"limit":100000}}'
# PUBLIC_API_TOKEN = ""  # 或通过 wrangler secret put
```

**Secrets（非 vars）**

```bash
wrangler secret put PASSWORD
wrangler secret put PUBLIC_API_TOKEN   # 可选
```

```env
PASSWORD=local-dev-password
```

## 8. 环境变量一览

| 变量 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `PASSWORD` | Secret | 空 | 未设置 = Dev 模式 |
| `USERNAME` | Var | admin | 内部会话标识（登录页不展示） |
| `ALERT_THRESHOLD` | Var | 70 | 告警阈值 % |
| `FREE_TIER_LIMITS` | Var | 内置 | JSON 覆盖限额 |
| `WEBHOOK_URL` | Var | 空 | 无 KV 渠道时的企微回退 |
| `ACCOUNT_CHECK_INTERVAL_MINUTES` | Var | 20 | 刷新间隔缓存 |
| `MAX_EXTERNAL_SUBREQUESTS_PER_RUN` | Var | 50 | 单次 subrequest 上限 |
| `PUBLIC_API_TOKEN` | Secret/Var | HMAC | 公开 API token |

## 9. 安全与约束

- **API Token**：被监控账号只读权限；存 KV；响应掩码
- **错误处理**：Token 失效 → `status: "error"`；D1/KV/R2/Vectorize 单组失败 → 对应指标 `available: false`
- **Session**：Web Crypto UUID + KV TTL 24h；HttpOnly Secure Cookie
- **禁止**：`wrangler.toml [env]` 多账号；后端硬编码 Account ID

### 10.2 Wrangler CLI

```bash
cd worker
npm install
npx wrangler kv namespace create KV
# 编辑 wrangler.toml 或使用 wrangler.deploy.toml（--config）
npx wrangler secret put PASSWORD
npm run typecheck && npm run deploy
```

本地开发：`worker/.dev.vars` → `PASSWORD=...` → `npm run dev`（:8787）

### 10.3 GitHub Actions

文件：`.github/workflows/deploy.yml`  
触发：`push` → `master` 或 `workflow_dispatch`

| Repository Secret | 必须 | 说明 |
|-------------------|------|------|
| `CLOUDFLARE_API_TOKEN` | ✅ | Workers + KV Edit |
| `CLOUDFLARE_ACCOUNT_ID` | ✅ | 托管账号 ID |
| `KV_NAMESPACE_ID` | ⚪ | 未设则 CI 查找/创建 title=`KV` 的命名空间 |
| `PASSWORD` | ⚪ | 设则 CI 执行 `wrangler secret put PASSWORD` |

CI 流程：`npm ci` → typecheck → sed 替换 KV id → `wrangler deploy` → 可选同步 PASSWORD。

### 10.4 wrangler.deploy.toml 模式

复制 `wrangler.toml` 为 `wrangler.deploy.toml`，填入真实 KV id，部署时：

```bash
npx wrangler deploy --config wrangler.deploy.toml
```

勿将含真实 id 的文件提交到公开仓库。

## 11. API Token 推荐权限

**必需**：

- Account Analytics: Read（GraphQL 用量，含 D1 读/写/存储）
- D1: Read（REST 列举 D1 数据库个数；缺此项时 D1 卡片显示权限提示）

**建议**：

- Cloudflare Pages: Read
- Workers Scripts: Read
- Workers KV Storage: Read
- Workers R2 Storage: Read
- Queues: Read
- Hyperdrive: Read
